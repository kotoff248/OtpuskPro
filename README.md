# Kabinet.pro

`Kabinet.pro` — веб-приложение на Django для управления отпусками,
согласованиями и годовым планированием отсутствий сотрудников.

Система ориентирована на HR, руководителей подразделений и сотрудников. Она
помогает собрать пожелания по отпускам, сформировать проект графика, проверить
его по кадровым и календарным ограничениям, согласовать график по маршруту
ответственных лиц и вести дальнейшие заявки, переносы и аналитику.

## Что умеет система

- личные кабинеты для сотрудников, HR, руководителей отделов, руководителя
  предприятия и уполномоченного согласующего лица;
- справочник сотрудников, отделов и управленческой структуры;
- календарь отпусков с отображением графика, заявок, переносов и отсутствий;
- заявки на оплачиваемый, неоплачиваемый и учебный отпуск;
- расчет оплачиваемого остатка по рабочим годам с учетом российских
  праздничных дней;
- сбор пожеланий сотрудников перед планированием годового графика;
- создание черновика графика отпусков и автоматическое действие
  `Добрать незакрытые дни`;
- проверка периодов по жестким бизнес-правилам: баланс, пересечения,
  минимальные непрерывные части отпуска, кадровое покрытие и риски отдела;
- нейромодуль поддержки решений для ранжирования допустимых вариантов отпуска;
- маршрут согласования графика: HR формирует проект, руководители отделов
  проверяют свои подразделения, затем график передается на финальное
  утверждение;
- согласование обычных заявок и переносов утвержденного графика;
- ручной сценарий закрытия неиспользованных дней прошлых периодов;
- уведомления, доска заявок и аналитическая панель для контроля нагрузки.

## Технологии

- Python
- Django 5.0.6
- PostgreSQL
- psycopg 3
- holidays
- NumPy
- PyTorch для обучения моделей
- JSON-артефакты моделей для runtime-скоринга
- Django Templates
- Vanilla JavaScript
- CSS без отдельного frontend-фреймворка

Обычная работа страниц не требует PyTorch-инференса. Обучение выполняется
отдельными management-командами, а приложение использует сохраненные JSON-модели.

## Роли пользователей

- `employee` — сотрудник: видит свой кабинет, календарь, заявки и пожелания.
- `hr` — HR: ведет сотрудников, планирование, черновики графика и заявки.
- `department_head` — руководитель отдела: согласует заявки сотрудников и
  проверяет график своего подразделения.
- `enterprise_head` — руководитель предприятия: принимает решения верхнего
  уровня и участвует в финальном согласовании.
- `authorized_person` — уполномоченное лицо для отдельных согласований.

Пользователь не может согласовать собственную заявку. Правила маршрутизации
сосредоточены в сервисном слое, а не в шаблонах.

## Структура проекта

```text
apps/
  accounts/      вход, выход, роли и auth-вспомогательная логика
  core/          уведомления, общие модели, demo/reset-команды
  employees/     сотрудники, отделы, профили и кадровое покрытие
  leave/         отпуска, графики, заявки, согласования, аналитика и ML
config/          настройки Django, корневые URL, ASGI/WSGI
scripts/         локальные helper-скрипты
static/          CSS и JavaScript приложения
templates/       Django-шаблоны и reusable partials
```

Ключевая бизнес-логика отпусков находится в `apps/leave/services`. Логика
планирования черновика разложена в `apps/leave/services/schedule_drafts`, а
модели ранжирования и обучение — в `apps/leave/ml`.

## Основные сущности

- `Employees` — сотрудник, роль, отдел, должность и связанный пользователь.
- `Departments` — подразделения и руководители.
- `VacationRequest` — заявка на отпуск или другое отсутствие.
- `VacationSchedule` — годовой график отпусков.
- `VacationScheduleItem` — конкретный период отпуска в графике.
- `VacationScheduleChangeRequest` — запрос переноса утвержденного периода.
- `VacationEntitlementPeriod` — рабочий год и положенные оплачиваемые дни.
- `VacationEntitlementAllocation` — списание дней отпуска по рабочим годам.
- `VacationPreferenceCollection` и `VacationPreference` — сбор пожеланий.
- `VacationScheduleCandidate` и `VacationScheduleCandidatePackage` — варианты,
  которые оцениваются при формировании черновика.
- `VacationScheduleDepartmentApproval` и `VacationScheduleEnterpriseApproval` —
  этапы согласования годового графика.

## Основные страницы

- `/` — вход в систему;
- `/main/` — личный кабинет;
- `/employees/` — сотрудники;
- `/departments/` — отделы;
- `/staffing/` — кадровое покрытие и demo-инструменты;
- `/calendar/` — календарь отпусков;
- `/calendar/planning/` — планирование годового графика;
- `/applications/` — заявки и переносы на согласовании;
- `/analytics/` — аналитика по отпускам и нагрузке;
- `/notifications/` — уведомления.

## Локальный запуск

Создайте `.env` по примеру `.env.example` и заполните параметры PostgreSQL:

```env
DB_NAME=your_database_name
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_HOST=localhost
DB_PORT=5432
```

Установите зависимости:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Примените миграции:

```powershell
.\.venv\Scripts\python.exe manage.py migrate
```

Запустите локальный сервер:

```powershell
.\scripts\django_server.ps1 -Action restart -Port 8001 -ReadyTimeoutSeconds 10
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8001/
```

Остановить сервер:

```powershell
.\scripts\django_server.ps1 -Action stop -Port 8001
```

## Demo-данные

Для локальной демонстрации можно пересоздать demo-БД:

```powershell
.\.venv\Scripts\python.exe manage.py seed_vacation_requests --confirm-reset --seed-value 42
```

Команда удаляет и заново создает demo-данные: сотрудников, отделы, графики,
заявки, переносы, уведомления и историю для обучения моделей.

Быстрый вариант заполнения:

```powershell
.\.venv\Scripts\python.exe manage.py seed_vacation_requests --confirm-reset --fast
```

## Нейромодуль

Нейромодуль используется как часть сценария формирования графика отпусков. Он не
заменяет правила предметной области: сначала система отбрасывает недопустимые
варианты по hard rules, затем ранжирует оставшиеся периоды и пакеты.

В проекте используются два уровня оценки:

- candidate scorer — оценивает отдельный период отпуска;
- package ranker — оценивает набор периодов сотрудника в годовом графике.

Обучение candidate-модели:

```powershell
.\.venv\Scripts\python.exe manage.py train_vacation_candidate_model
```

Обучение package-модели:

```powershell
.\.venv\Scripts\python.exe manage.py train_vacation_package_model
```

Для демонстрации можно выбрать версию scorer через `.env`:

```env
VACATION_CANDIDATE_SCORER_VERSION=vacation-candidate-mlp-v2
```

## Проверки

Базовая проверка Django:

```powershell
.\.venv\Scripts\python.exe manage.py check
```

Тесты основных приложений:

```powershell
.\.venv\Scripts\python.exe manage.py test apps.accounts apps.employees apps.leave apps.core --keepdb
```

Если тестовая база устарела после новых миграций, тесты можно повторить без
`--keepdb`.

## Статус проекта

Проект разрабатывается как демонстрационная система для магистерской работы.
Основной акцент сделан на управлении отпусками, кадровом планировании,
объяснимом формировании графика и проверяемом маршруте согласований.
